# AI Workflow Kit

This project uses a structured 4-phase development workflow managed through agent skills and commands.

## Workflow Phases

All development follows this pipeline:

1. **init-project** -- Generate project documentation (PRD, architecture, rules) through iterative Q&A
2. **plan-feature** -- Plan a feature with detailed spec, implementation plan, and acceptance tests
3. **exec-feature** -- Implement the feature until all acceptance tests pass
4. **embed-feature** -- Integrate the verified feature into project documentation and run integration tests

Invoke phases via `/init-project`, `/plan-feature`, `/exec-feature`, `/embed-feature`.

## Key Conventions

### Documentation

- `docs/prd.md` is a compact index. Never duplicate feature details there.
- `docs/features/*.md` are the source of truth for each feature's specification.
- Feature docs are permanent and evolve through the lifecycle (planned -> implemented -> integrated).

### Testing

- `*.acceptance.test.`* files are immutable contracts created during plan-feature. Never modify them during exec-feature.
- `*.test.`* files (without `.acceptance.`) are mutable implementation tests.
- `tests/integration/` and `tests/e2e/` are created during embed-feature for cross-feature concerns.

### Rules

- Domain-specific rules live in `.agents/rules/`. They are generated by init-project and updated by plan-feature and embed-feature as the project evolves.
